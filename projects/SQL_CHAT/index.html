Conversational SQL Agent

Summary

Engineered a full-stack, AI-powered conversational agent using Streamlit and LangChain/Groq to resolve the challenge of non-technical users accessing complex data (Situation/Task). This was accomplished by designing and implementing an abstracted, high-speed system that translates natural language into executable SQL, successfully supporting dynamic connections to both local SQLite and remote MySQL databases (Action). This democratized data access and achieved near real-time query results due to optimized LLM integration and resource caching (Result).

GitHub Repository

Approach & Core Functionality

  • Integrated the high-speed Groq LLM with a ZERO_SHOT_REACT_DESCRIPTION agent, optimizing the translation of diverse user queries into accurate, executable SQL code.
  • Developed a flexible and abstract database configuration pipeline, supporting seamless, dynamic connection switching between local SQLite files and remote MySQL servers using the SQLAlchemy engine.
  • Implemented Streamlit's @st.cache_resource decorator to streamline database connection initialization, enhancing application performance and resource efficiency across multiple user sessions.
  • Designed a transparent user experience by incorporating a StreamlitCallbackHandler to display the LLM's thought process (tool calls and generated SQL), building trust and improving debuggability.

Key Technologies

  • **Python & @dataclass:** Core programming language and defining the shared, structured state for data management.
  • **LangChain/Gemini:** Provides the Generative AI capabilities for analysis, test generation, and summarization.
  • **LangGraph:** The orchestration layer that defines the fixed, multi-step pipeline and manages the state flow between agents.
  • **Streamlit:** The user interface layer, allowing easy input of the PR URL and display of the final, formatted review report.
