Automated PR Review Agent: A Multi-Agent System with LangGraph & Gemini

Summary

Engineered a multi-agent, sequential pipeline using **LangGraph** to automate the Pull Request (PR) review process, **reducing manual code inspection time** and enhancing code quality by enforcing standardized checks for bugs, security, and performance.

GitHub Repository

Approach & Core Functionality

  • Architected a four-step agent workflow (Fetcher, Reviewer, Test Coverage, Summarizer) orchestrated by an **evolving Python dataclass state**, ensuring seamless and structured data flow across all stages.
  • Integrated the **Gemini** generative model to execute sophisticated tasks, including **deep code analysis** to identify subtle vulnerabilities and **dynamic generation** of missing `pytest` stubs for uncovered functions.
  • Developed a **Streamlit** front-end that accepts a GitHub PR URL and displays a comprehensive, natural language review report, making the complex GenAI-driven process accessible to developers.
  • Implemented a specialized **Code Review Agent** that performs preliminary static checks (e.g., risky keyword identification) before engaging the LLM for advanced analysis, **optimizing efficiency** and minimizing token usage.

Key Technologies

  • **Python & @dataclass:** Core programming language and defining the shared, structured state for data management.
  • **LangChain/Gemini:** Provides the Generative AI capabilities for analysis, test generation, and summarization.
  • **LangGraph:** The orchestration layer that defines the fixed, multi-step pipeline and manages the state flow between agents.
  • **Streamlit:** The user interface layer, allowing easy input of the PR URL and display of the final, formatted review report.
